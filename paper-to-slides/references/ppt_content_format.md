# `ppt_content.md` 格式规范

生成 `ppt_content.md` 时严格遵守这里的每一条。

## 核心原则

| 原则 | 说明 |
|------|------|
| **论文名片** | 目标是激发兴趣，而非详尽解读 |
| **聚焦创新** | 阐述方法创新与启示，避免罗列性能指标 |
| **结构灵活** | 根据论文特点调整逻辑流程，不拘泥固定模板 |

## 页数与篇幅

- **默认严格 2 页**：即便论文内容丰富，也压缩到 2 页，不得自行加页。内容多是"取舍"而非"加页"的理由。
- **仅当用户明确要求**"详细些/多页/N 页"时，才取 **2-5 页**（上限 5）。
- **加页怎么分配**（仅在用户要求加页时）：把所选逻辑流程的阶段铺到更多页，每页聚焦一个阶段（如 3 页 = Method / Results / Discussion）；可纳入更多支撑图表。
- 无论几页，编号跨页连续，所有页共用同一标题行。

## 硬性格式要求

| 项目 | 要求 |
|------|------|
| 标题 | 所有页共用论文完整标题 + 斜体来源（如 `*ICML 2023*`） |
| 大纲符号 | 一级 `▶`，二级 `▢`，三级 `-`（dash） |
| 编号 | 跨页连续，不重置（Slide 1 用 1.x、2.x；Slide 2 接 3.x…，依此类推） |
| 语言 | 幻灯片内容**全英文**，讲稿**中文** |
| 强调 | 用 `**粗体**`、`*斜体*`、`` `代码样式` `` 突出关键概念 |

## 每页行数预算（布局约束，必须遵守）

内容超出会溢出幻灯片边界，所以每页有行数上限。预算取决于该页是否带标题：**首页**顶部要放论文标题+来源，可用空间少；**后续页**是空白模板页，可用空间多。

统计规则：**仅计算以 `▶`、`▢`、`-` 开头的内容行**（不含空行、`**配图**:`、`**讲稿**:` 等）。

| 页类型 | 总行数范围 |
|--------|-----------|
| **首页**（Slide 1，带标题） | **15 ~ 18 行** |
| **后续每页**（Slide 2-5，无标题） | **17 ~ 21 行** |

草稿写完后**必须逐页数行**；超限时优先删减低价值的三级 `-` 行，保留核心论点。

## 内容组织

- **每页结构**：2-4 个一级论点，每个下设 1-3 个二级论点。
- **逻辑流程**（按论文类型择一）：
  - `Background/Motivation → Method → Results → Conclusion`
  - `Problem → Framework/Architecture → Innovations → Results`
  - `Inspiration → Dataset → Methods → Results`
  - `Key Contributions → Training Innovation → Achievements`

## 配图配置

- **位置**：幻灯片右侧（构建器默认排在右侧区域，宽度可在排版精修时调整）。
- **数量**：每页 1-4 张。
- **内容**：架构图、流程图、关键结果对比图/表。
- **引用格式**：必须与论文原文**完全一致**（大小写、空格、括号），因为下游提取与文件名匹配依赖它。

| 引用类型 | 示例 | 说明 |
|----------|------|------|
| 完整图表 | `Figure 1`, `Table 2` | 提取整个图表（含所有子图） |
| 子图 | `Figure 1(a)`, `Figure 1(b)` | 仅提取复合图中的特定子图 |
| 简写 | `Fig. 1`, `Fig 1` | 等同 `Figure 1` |

### 提取模式标记

| 模式 | 语法 | 内容 | 适用 |
|------|------|------|------|
| 完整模式 | `Figure 1` | 含标题、图例、注释 | 图表自解释、讲稿不细讲 |
| 主体模式 | `Figure 1 [content-only]` | 仅图表主体，不含标题说明 | 讲稿会用中文详解、避免中英重复 |

选择原则：讲稿会详细解释 → 用 `[content-only]`；需保留原文说明 → 用完整模式；子图通常用完整模式（子图标签 `(a)` 是必要的）。

## `ppt_content.md` 标准模板

```markdown
# [论文完整标题]
*[经检索确认的精准来源，如：ICML 2023 / Nature Communications 2026 / arXiv 2026]*

---

## Slide 1

▶ **1. 一级论点**

▢ 1.1 二级论点
  - 具体描述（全英文）
  - 具体描述

▢ 1.2 二级论点
  - 具体描述

▶ **2. 一级论点**

▢ 2.1 二级论点
  - 具体描述

**配图**:
- Figure 1 [content-only]
- Table 2

**讲稿**:
（中文讲稿，帮助汇报者口头解释本页内容）

---

## Slide 2

▶ **3. 一级论点**

▢ 3.1 二级论点
  - 具体描述

**配图**:
- Figure 3(a)
- Figure 3(b) [content-only]

**讲稿**:
（中文讲稿）
```

### 配图列表解析规则（供下游对齐）

- 每个图表占一行，以 `- ` 开头。
- 名称支持 `Figure N` / `Fig. N` / `Table N` / `Figure N(x)`。
- 行内 `# ...` 之后视为注释，会被忽略。
- `[content-only]` 标记 → 主体模式；无标记 → 完整模式。

## `paper_summary.md` 要求

详尽客观的**中文**论文总结，为汇报者讲解论文服务，建议 5000 字以上，覆盖：研究背景与动机、核心问题、方法/架构细节、关键创新点、实验设置与主要结果、局限与启示。

## 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 论文内容单薄 | 保持默认 2 页，扩展背景或讨论启示，不强行加页 |
| 图表质量差/缺失 | 配图位置注明 `[建议重绘: Figure X]` 或 `[无合适配图]` |
| 综述类论文 | 聚焦分类框架与关键洞见，而非单一方法 |
| 多个并列贡献 | 优先选最具创新性的 1-2 个深入介绍 |
| 复合图需部分展示 | 用子图引用 `Figure 1(a)` 而非整图 |
| 需中文详解图表 | 用 `[content-only]`，避免中英文说明重复 |
