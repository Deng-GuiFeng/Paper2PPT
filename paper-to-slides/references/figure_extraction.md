# 图表提取规范

图表通过查看渲染出的页面图、判断裁剪范围来提取：脚本负责文本扫描定位候选页、渲染、裁剪等确定性操作，**看渲染出的页面图给出 bbox 坐标**，再裁剪、对照校验图自检并精修。

## 整体流程（每个图表循环）

对 `ppt_content.md` 中 `**配图**:` 列出的每个图表，执行：

1. **定位 + 渲染**
   ```bash
   python $SKILL/scripts/locate_figure.py "<pdf_path>" "<figure_name>" -d "<work>/pages" --dpi 200
   ```
   脚本做一次 O(n) 文本扫描，返回候选页（1-indexed，末次出现优先），并把候选页渲染成两张图：
   - `_<name>_p<N>.png` —— **干净图，用于裁剪**
   - `_<name>_p<N>_grid.png` —— **叠加 0-1000 坐标网格，供你查看读坐标**

2. **看图判断真正含图的页**
   查看 `*_grid.png`。注意：文本扫描会命中**正文里的引用**（如 "as shown in Figure 1"），那一页往往不含图本身。逐个候选页看，找到真正画着该图表的那页。若候选页都不含图（或 `fallback=true`），用 `--max-pages` 调大重渲，或直接渲染相邻页。

3. **读出 bbox（0-1000 坐标）**
   对照网格读出目标的 `x1,y1,x2,y2`：左上角 `(0,0)`，右下角 `(1000,1000)`。按下面的「模式规则」决定边界含不含标题。

4. **裁剪（同时生成校验图）**
   ```bash
   python $SKILL/scripts/crop_figure.py "<work>/pages/_<name>_p<N>.png" \
       --bbox x1,y1,x2,y2 -o "<work>/asset/<OutputName>.png"
   # 子图加 --sub（更紧的 padding）
   ```
   脚本除了输出裁剪图，还会生成一张 `<OutputName>_check.png` —— 把你选的框**画在带网格的整页上**。

5. **看校验图核对边界 + 精修（关键，多轮）**
   **务必查看那张 `*_check.png`**：蓝框直接画在页面上，能一眼看出某条边是否切到内容、是否带进了下一段正文/标题/相邻图表。这比只看裁剪结果可靠得多（裁剪图看起来"完整"时，校验图能暴露出底部少了一行、或多带了一个标题）。
   不满意就调 bbox 重裁，循环到框线贴合为止（约 ≤3 轮）。常见修法：标题被切 → 扩对应边；带进无关文字/下一节标题 → 收紧对应边；表格末行/某列被切 → 适当外扩。
   满意后可再看一眼裁剪图 `<OutputName>.png` 终检。

## 完整示例

以提取 `Figure 1`（完整模式，一张架构图）为例，展示一轮精修：

1. 定位：`python $SKILL/scripts/locate_figure.py paper.pdf "Figure 1" -d work/pages` → 候选页 `[5, 2]`。
2. 看图：查看 `work/pages/_Figure_1_p5_grid.png` 发现第 5 页只有正文里"...as illustrated in Figure 1..."的**引用**，没有图；改看 `_Figure_1_p2_grid.png`，第 2 页才是真正的架构图，下方有标题"Figure 1: ...".
3. 读 bbox：对照网格，图体加下方标题约 x[120,885]、y[560,880]。
4. 裁剪：`python $SKILL/scripts/crop_figure.py work/pages/_Figure_1_p2.png --bbox 120,560,885,880 -o work/asset/Figure_1.png`。
5. 看校验图 `Figure_1_check.png`：发现蓝框顶边带进了上一行正文 → 把 y1 从 560 下移到约 580 排除该行 → 重裁 `--bbox 120,580,885,880`。再看校验图，框线贴合，定稿。

要点：候选页可能只是"引用页"（步骤 2），以及"看起来完整的裁剪图也可能多带一行"（步骤 5）——这两点都靠看图发现，不能只凭坐标。

## 模式规则：边界该含什么

### 完整模式（默认，对应无标记的 `Figure 1`）

**必须包含完整标题**（最常见错误是漏掉标题）：
- 图（Figure）：标题在**图下方**，找以 `Figure N:` 或 `Figure N |` 开头的文字，常跨 2-3 行，全部包含。
- 表（Table）：标题在**表上方**。
- 同时包含：主体图/表、图例与色键、坐标轴标签、复合图的子图标签 (a)(b)(c)、属于本图的注释脚注。
- 宁可多含一点也不要切掉标题文字；标题下方留一点余量。

### 主体模式（对应 `[content-only]`）

只要**主体视觉内容**（裁剪时把 bbox 收到不含标题即可，无需额外参数）：
- 包含：图/表数据本身、作为视觉一部分的坐标轴标签、嵌入图内的图例、复合图各子面板。
- **不包含**：图号与标题文字（`Figure 1: ...`）、外部图例/色键、脚注与来源说明、任何描述性而非图体的文字。

### 子图（`Figure 1(a)` 等）

- 只框定该子面板本身，含其自身标签 `(a)`。
- 不要含主标题或其他子图。
- 裁剪时加 `--sub`。

## 质检评分（自检用，越严越好）

裁完看一眼，对照打分，<10 分就精修：

**完整模式**：
- 10 = 主体齐全 + 标题完整 + 无多余内容
- 8-9 = 边界略不精确
- 6-7 = 标题被部分切掉 / 缺图例
- ≤5 = 严重问题（切掉主体、混入其他图表）
- 重点检查：整段标题（可能多行）是否都在框内？主体是否完整？是否误含**其他**图表或无关正文？

**主体模式**：
- 10 = 仅主体、干净、无标题
- 8-9 = 轻微边界问题
- 5-7 = 含进了标题 / 漏内容
- ≤4 = 严重错误

**子图**：
- 10 = 仅该子图、边界正确、含自身标签
- 5-7 = 含进相邻子图或漏内容
- ≤4 = 框错子图

收敛判据：连续两轮 bbox 几乎不变（各边变化 < 5 单位）即可定稿。

## 输出命名（必须严格，下游构建器靠它匹配）

把图表名里的空格换成下划线，括号保留：

| `ppt_content.md` 引用 | 输出文件名 |
|----------------------|-----------|
| `Figure 1` | `Figure_1.png` |
| `Figure 1 [content-only]` | `Figure_1.png`（同名，模式只影响裁剪范围） |
| `Figure 1(a)` | `Figure_1(a).png` |
| `Table 2` | `Table_2.png` |

全部存入同一个 `asset/` 目录。`build_ppt.py` 的 `find_figure_file` 会做模糊匹配（空格↔下划线、大小写、子串），但按上表命名最稳。

## 异常处理

| 问题 | 处理 |
|------|------|
| 文本扫描未命中（扫描件/无文字层）| 脚本返回 `fallback=true` 并给出前几页；逐页查看、人工找图 |
| 候选页只有引用、不含图 | 看其他候选页；必要时渲染相邻页 |
| 图太密/裁出来糊 | `locate_figure.py` 加 `--dpi 300`（甚至 400）重渲再裁 |
| 实在提取不到 | 跳过并在最终报告中注明，建议用户手动替换 |
